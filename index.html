Tech JQuery, Ajax
Database MariaDB

TOOL Eclipse(Maven), HeidiSQL, VSCode

사용기술
  1. AOP
    • 사용한 이유
      - 컨트롤러에서 메서드를 실행하기 전에 관리자 외 사용자의 접근을 막기 위한 조건 검사 코드의 중복을 피하기 위해
      - AOP를 활용하여 해당 조건 검사를 모듈화한 Advice를 Aspect로 분리하여 핵심 로직과 분리함
  2. Jquery(AJAX)
    • 사용한 이유
      - 중복된 데이터 입력을 실시간으로 확인하여 방지하기 위해
      - 특정 영역의 페이징 처리 및 로드를 동적으로 구현하기 위해
      - 주문 및 환불 상태 변경과 같은 작업을 비동기적으로 처리하여 페이지 전체를 다시 로드하지 않기 위해
담당 업무
담당 업무 : DB설계, 관리자 페이지(전체), 사용자 페이지(고객센터, 챌린지 이벤트)


  1. 관리자 페이지
    • 메인 페이지(AJAX)
      - 오늘 날짜를 기준으로 주문 상태에 따른 건 수를 가져와서 JSON 객체로 반환하여 표시함
      - 최근 7일간의 매출 금액, 취소 금액, 환불 금액 및 해당 건 수를 가져와서 JSON 객체로 반환 후, 차트와 테이블로 표시함
    • 페이징 및 검색
      - 모든 관리 목록 페이지에서 페이징 알고리즘을 사용 (이전 : -1 페이지, 다음 : +5 페이지)
      - 검색 기능과 필터링은 마이바티스 동적 쿼리를 사용
    • (다중) 이미지 파일 업로드
      - Multipart 리스트로 다중 파일 이미지를 받아와서, 반복문을 돌며 서버 경로에 각각의 파일을 업로드
      - DTO Setter 메서드에서 파일명들을 콤마(,)를 구분자로 하여 문자열로 저장
      - CK에디터를 사용하여 상세설명 이미지 업로드
    • @Transactional 어노테이션을 사용한 트랜잭션 처리 (AJAX)
      - 포인트 지급 : 1. 회원 포인트 update 2. 지급 내역 insert 3. 지급 상태 update
      - 입고 수정 : 1. 제품 수량 update 2. 입고 내역 update
    • 메일
      - mail.properties 파일과 javax.mail, javax.activation 라이브러리를 활용하여 문의 답변 시 이메일을 보냄
    • 날짜 범위검색을 통한 매출 파악
      - 리스트에서 주문 상태와 환불 상태 금액의 합을 구하고, 범위 내의 일수로 나누어 평균을 계산함
    • FAQ게시판 (AJAX)
      - @RESTController 어노테이션을 사용해서 CRUD 기능을 하나의 .jsp 파일에 AJAX를 사용하여 처리
문제 발생과 해결 과정
  1. AJAX를 사용한 주문상태 변경
    • 문제
      - 상태 변경 버튼을 누르면 DB의 데이터는 변경되지만, View 페이지에서는 데이터가 공백으로 보이거나 해당 상태의 카테고리로 이동하지 않는 문제가 발생
    • 해결
      - 변경된 POST 요청이 성공한 후에 상태 리스트를 호출하는 함수가 없었기 때문에, 로드 함수를 분리하여 개별적인 함수를 만듦
      - 해당 상태의 카테고리로 이동하기 위해 location.reload() 함수를 사용
  2. 다중 이미지 파일 수정
    • 문제
      - 제품 수정 폼으로 이동한 후 아무 작업을 하지 않고 수정을 위해 POST 요청을 보냈을 때, 제품 상세 이미지가 사라지고 데이터베이스(DB)에는 공백이 들어가는 문제가 발생
    • 문제점 발견
      - 히든 태그에 이미지 파일 이름을 콤마(,)로 구분한 문자열로 저장했으나, POST 요청 시 다중 파일 입력태그가 null 값으로 전송되어 setter 메서드에서 공백으로 설정되어 업데이트되는 문제점을 발견
    • 해결
      - 컨트롤러에서 히든 태그의 값을 다시 별도로 받아와, 다중 파일 입력태그의 값이 null일 때 원래의 값을 설정하기 위해 set 메서드를 사용하여 업데이트

MVC 모델2를 사용해 Java와 HTML 파일을 분리하여 커뮤니티 사이트를 구현하는 팀프로젝트를 진행했습니다.
게시판과 회원관리를 중점으로 팀원과의 역할 배분을 하여 진행했습니다. 		Language Java, HTML5, CSS3, JavaScript
Tech JQuery, Ajax
Database MariaDB
FrameWork MyBatis, Bootstrap
TOOL Eclipse, HeidiSQL, VSCode

Github
사용기술
  1. Jquery(AJAX)
    • 사용한 이유
      - 추천 : 사용자가 게시물을 추천할 때, 페이지 전체를 새로고침하지 않고 추천 카운트를 업데이트하고 싶었기 때문
      - 댓글 페이징 : 많은 댓글이 있는 게시물의 페이지 로딩 속도를 개선하기 위해 동적으로 댓글 목록을 로드하기 위함
  2. 크롤링
    • 사용한 이유
      - 특정 웹사이트에서 발매되는 상품 정보를 실시간으로 모니터링하고자 크롤링 기술을 사용하였음
담당 업무
담당 업무 : DB설계, 전체 페이지 디자인, 커뮤니티 게시판, 댓글, 크롤링


  1. 게시판
    • 페이징 및 검색
      - 모든 목록 페이지에서 페이징 알고리즘을 사용 (이전 : -1, 다음 : +5)
      - 검색 기능과 필터링은 마이바티스 동적 쿼리를 사용
    • 게시글 공개/비공개
      1. 히든 태그에 해당 페이지 게시물 목록 번호를 공백을 기준으로 하여 저장
      2. POST 요청으로 히든 태그 값과 체크된 게시물 번호를 서버로 전송
      3. 전체 게시물 번호와 체크된 게시물 번호를 공백을 기준으로 split하여 List에 저장
      4. removeAll 메서드를 사용하여 체크되지 않은 게시물 번호의 List를 구한 후 update
    • 이전/다음글
      - 서브쿼리를 사용하여 현재 게시물 번호의 다음글/이전글 번호를 가져와 href 링크로 설정함
    • 게시글 추천
      - AJAX를 사용하여 게시물 번호와 회원의 닉네임을 파라미터로 받아와 추천 테이블에 insert
      - AJAX를 사용하여 페이지 로드 시, 해당 게시물을 추천한 회원의 경우 추천 버튼이 활성화
    • 댓글작성/댓글추천
      - 댓글 작성과 추천 시 AJAX를 사용하여 페이지 이동 없이 본인이 작성한 위치에서 바로 보이게 하였음
  2. 발매정보/발매소식
    • jsoup 크롤링
      - '럭키드로우'라는 실시간 발매정보 홈페이지를 크롤링하여 필요한 정보의 태그와 클래스명을 파싱하여 객체에 저장
    • 검색
      - 크롤링하는 과정에서 String 변수의 contains 메서드를 사용하여 제목과 발매처를 파라미터로 받아 필터링
문제 발생과 해결 과정
  1. 첫번째 댓글 작성
    • 문제
      - 모든 게시물의 첫번째 댓글 작성 시 그 댓글은 보이지 않고 두 번째 댓글 작성 부터 보이는 현상
    • 문제점 발견
      - 댓글 작성 시 댓글 목록을 날짜 오름차순으로 정렬하여 맨 뒷페이지로 작성한 댓글을 빠르게 확인할 수 있도록 구현 하였음
      - 이를 위해 히든 태그에 젤 마지막 페이지 번호를 저장하고, AJAX를 사용하여 목록 페이지를 마지막 페이지로 이동 시켰음
      - 그러나 페이징 알고리즘에서 첫 번째 댓글을 작성하기 전에 댓글 개수가 0이므로 페이지 번호가 0이 되어 쿼리문에서 null이 되어 아무 내용도 보이지 않게 됨
    • 해결
      - 댓글 작성 요청을 AJAX로 보내고, 마지막 페이지 번호의 히든 태그 ID 값이 0인 경우 1로 설정하여 문제를 해결하였음
